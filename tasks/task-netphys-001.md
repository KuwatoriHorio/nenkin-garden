## goal
一般グラフ上で「扇状探索 → 衝突で網化 → 周期 consolidation（Kirchhoff＋Tero 刈り込み・最外周を新前線）

## モデル仕様（設計メモ §1〜§3・担体A）
- **状態（一般グラフ）**: ノード `{id,x,y}`、辺 `{a,b,D,L}`（無向・D=コンダクタンス、L=長さ、**ループ可**）。
  **質量 = `free_budget + Σ(D_e·L_e)`**（担体A：構造質量は辺に載る）。回収で増、探索で free→辺、刈り込みで辺→free。
  アクティブ前線ノード集合。砂糖（置く/取る）は sink/source。場（陸海・E・`default_home`）は共有・読み取り。
- **`netphys_step`（決定論・id 昇順・単一 PRNG）**:
  - **Phase1 探索（毎tick）**: 各前線ノードから space colonization＋ランダム探索（tree-growth-002 の w_rand 発想）で
    ノード/辺を外へ追加（新辺 D 小）。free_budget を消費。海/範囲外は棄却、標高でコスト増。
  - **Phase2 anastomosis（探索中）**: 成長ノードが既存ノード/辺の**融合距離 d 以内**に来たら辺を追加してループを閉じる
    （グリッド近接で決定的検出）。
  - **Phase3 consolidation（周期 N tick ごと）**: (1) **最外周ノード**（重心から遠い上位 k）を新成長点に選ぶ。
    (2) 最外周点＋残存砂糖を source/sink に **Kirchhoff を解いて**各辺 Q_e を得る（標高は `L_eff=L·(1+α·meanE)`）。
    (3) **Tero 適応**: `dD = f(|Q|)−D` 相当で流量多い辺は D 増・少ない辺は D 減、D<ε の辺を除去、
    生存経路に無い孤立/行き止まりノードを prune（＝経路を残して縮退）。刈った `Σ D·L` は free_budget へ戻す。
    (4) 生存最外周ノードが次の前線。
  - **Phase4 移動**: 後方を刈って得た体積で前線が再拡散し重心が外へ動く（前進波）。
- **Kirchhoff の再利用**: `analysis::flow` の Kirchhoff/線形ソルバを**読み取り再利用**する。もし関数化・可視性調整が
  要るなら **analysis の挙動・ゴールデンを変えない範囲で加算的に**（例: 純粋な dense ソルバを pub 化する薄いラッパ）。
  それが困難なら netphys 内に**最小の決定論 dense Kirchhoff** を実装してよい。**いずれも analysis テストは緑のまま**。
- **パラメータ** 新規 `NetParams`（現行 `Params`/`TreeParams` を汚さない）: 周期 N・融合距離 d・最外周点数 k・
  Tero 強化/減衰・prune しきい ε・探索率・標高 α・node/edge cap・初期予算 等。既定値・具体数は実装者裁量（実測で調整）。

## 受け入れの段階化（ユーザー確定・2026-07-11）
本モデルは全条件を同時に満たすのが難しいため**段階的**にする。
- **本タスク netphys-001 = Stage 1（合否対象）**: ①②⑤⑥。Phase 1〜3（扇状探索 → 衝突で網化 →
  周期 consolidation で Kirchhoff＋Tero 刈り込み・最外周を端子に）を実装し、**餌を結ぶ連結網**を
  保存則・決定性・有界のもとで作る。**Phase 4 の"歩く"は最小限あってよいが合否に問わない。**
- **繰り延べ → netphys-002（本タスクの合否対象外）**: ③前進波で有意に移動・④consolidation で効率改善。

## acceptance_test（Stage 1・headless・複数シード中央値・S9 部分集合 [1,42,1337]）
1. **網化**: フロントが衝突する配置で、少なくとも1つの連結成分が**ループを持つ**（辺数 > ノード数−1＝冗長度>1）。
2. **餌を結ぶ**: 砂糖を2箇所以上置くと consolidation 後に**同一連結成分**で結ばれ `flow_connected`＝true・有限コンダクタンス。
5. **不変条件**: 保存則 `total_mass == collected − consumed`・非負・free 超過なし／境界（全ノード陸内）／
   再現性（同一 seed・入力列 → 同一 `netphys_state_hash`）／ソフト標高忌避（高標高帯の辺密度 < 低標高帯、
   または高標高辺が優先的に刈られる）。
6. **有界（性能）**: node/edge 数が cap 内に収まる（爆発しない）。周期 Kirchhoff の総コストが予算内。

3. **前進波で移動**: K サイクル後にコロニー重心が初期から**有意に外へ移動**（中央値でしきい超）。
4. **効率化（Tero）**: consolidation 前後で端子間の総コンダクタンス／`transport_efficiency` が**改善**（背骨へ刈り込み）。

## constraints
- **編集可**: 新規 `src/netphys/*`、`src/bin/` の新規ヘッドレス、新規 `tests/netphys_001.rs`、`lib.rs` への
  モジュール登録、`analysis::flow` の**加算的な可視性調整のみ**（挙動・ゴールデン不変）、`world.rs` の読み取り利用。
- **編集不可（人間所有・§7/§11・フック保護）**: Jones/tree の不変条件・ゴールデン・受け入れテスト、
  `tests/test_harness_001.rs`/`tests/metric_thresholds_001.rs` の baseline、analysis の受け入れアサート、規約・設計軸（§0）。
  **現行 Jones・tree のモデルコードを変更しない**（並置）。
- **決定性**: 探索は単一 PRNG・id 昇順。Kirchhoff/刈り込みは乱数不要で決定的。壁時計等を使わない。
- **ソフトのみ・壁禁止**: 標高はコスト/実効長で表現。**保存則を壊さない**（融合/刈り込み/非放射伸長の会計をテストで締める）。
- **過学習禁止**: 単一シードだけで通る調整は不可。複数シード中央値で判定。§7: 通すために tick/seed/解像度/cap を都合よく縮小しない。

## §8 停止の指針（このタスク特有・重要）
本モデルは「網化・餌連結・前進波移動・効率改善・保存/決定性・有界」を**同時に**満たすのが本質的に難しい。
最小構成で受け入れ①〜⑥を素直に満たせない設計上の壁（例: 前進波が安定移動せず its場で脈動する／爆発 vs 枯渇の
甘い帯が無い／保存則が締まらない）に当たったら、**§7 のごまかしをせず停止し、どの部分集合が成立し何が詰まったか・
選択肢**を報告する（nenkin-escalation 相当）。無理に全条件を通すために cap/tick を縮小しない。
