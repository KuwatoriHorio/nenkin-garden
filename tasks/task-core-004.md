## 背景
現状 trail 場に上限が無いため、エージェントが集まる場所ほど deposit で trail が濃くなり、
感知でさらに集まる**正のフィードバック**が暴走する（[step.rs](../src/step.rs) 拡散・減衰節）。
core-002 のコホージョン（空白へ出にくい）がこれを増幅し、時間が進むとホーム中心に
**深い誘引の井戸**ができて群れが局在化する。離れた砂糖のビーコン（固定強度 `sugar_beacon`）は
蓄積したホーム trail に勝てず、砂糖を置いても群れが離れない。

ユーザー要望（2026-07-07）: trail 濃度に**上限**を設け、過度な局在化を防ぎたい
（砂糖を置いたらコロニーから離れて探しに行けるように）。

## task_id
`core-004`

## goal
コアに **trail 濃度の上限 `trail_max`** を追加し、有効時は各 tick の trail 更新後に
`trail[i]` を `trail_max` でクランプする。これによりホームの誘引の井戸が頭打ちになり、
遠くの砂糖ビーコンが対等に競えて群れの局在化が緩和される。**既定は上限なし（現行挙動を維持）**。

## 実装方針
- **パラメータ**（[params.rs](../src/params.rs)）: `pub trail_max: f64`、既定 `f64::INFINITY`（＝上限なし・
  `v.min(INF)==v` で挙動バイト不変。分岐不要）。
- **クランプ**（[step.rs](../src/step.rs) 拡散・減衰の最終書き戻し）: `state.trail[i] = (v as f32).min(trail_max as f32)`。
  海マスク（v=0）は従来どおり。ソフトな飽和であって壁ではない（値を止めるだけで移動を禁じない）。
- 決定論・単一PRNG・index昇順・スナップショット感知を崩さない。クランプは決定的。

## acceptance_test（headless で判定可能・複数シード中央値）
検証は正準9シード S9 の部分集合（最低 `[1,42,1337]`）。foraging プリセット
（`init_cluster_sigma=3, w_trail_cohesion=1, home auto`）で比較する。

1. **局在化の緩和（主目的）**: 十分長い実行（例 T=300tick）後、**局在化指標**
   L = 「ホーム半径 R 内の trail 総量 / 全 trail 総量」を、有限 `trail_max`（例: `sugar_beacon` の
   数倍程度）と `trail_max=∞`（既定）で比較し、**有限上限のほうが L が有意に小さい**
   （群れ/網がホームに偏りにくい）を複数シード中央値で示す。R・具体しきいは実測で定める。
2. **探餌の改善（機能）**: ホームから離した砂糖を置いたとき、有限 `trail_max` のほうが
   その砂糖の回収量（`collected_total` の当該分）が同等以上〜多い（群れが離れて到達しやすい）。
   複数シード中央値で退行しない。
3. **上限の実効**: 有効時、全 trail セルが `<= trail_max + ε` に収まる。
4. **不変条件維持**: 有限性・境界・保存則・再現性・ソフト標高忌避（mean_hi<mean_lo）を S9 で維持。
5. **リグレッション無し（最重要）**: 既定（`trail_max=∞`）で `core-000/001/002/003`・`analysis-00x`・
   `test_harness_001`・`metric_thresholds_001` が**全て緑・final_state_hash 不変**（`v.min(∞)==v`）。
   もし既定で hash が変わるなら**停止して §8 で相談**（保護ゴールデンは編集しない）。
6. **決定性**: 有限 `trail_max` でも同一 (seed, script, ticks) → 同一 final_state_hash。
