## 背景
現状の木モデルは**誘引点（近くの砂糖）方向にしか伸びず**、砂糖が `attract_radius` 圏外だと成長せず
退縮する（[src/tree/step.rs](../src/tree/step.rs)）。ユーザー要望（2026-07-09）:
- **砂糖が無くてもランダムに伸長**（探索的に彷徨って広がる）してほしい。
- **砂糖が近くにあれば誘引**される（＝現行挙動）ことは保つ。

feasibility 検討の結論（矛盾なし）: 成長方向を**重み付きブレンド**
`dir = normalize(w_rand·ランダム単位ベクトル + Σ 誘引_i·重み_i)` にすれば、砂糖が無い/遠いと
ランダムが支配（探索）、近いと誘引が支配（配向）、と連続的に両立する。唯一の注意点は**保存則**:
ランダム伸長も体積予算 B を消費させ、**予算が尽きたら止まる**こと（無から無限成長は不可）。

## task_id
`tree-growth-002`

## goal
木モデル（`src/tree/`）の `tree_step` に**ランダム探索伸長**を追加し、砂糖が無くても手持ちの予算 B の
範囲で tip がランダムに伸長・分岐して広がり、近くに砂糖があれば誘引が支配して到達するようにする。
**既定は現行挙動（探索オフ）**とし、探索は新パラメータ `w_rand>0` で有効化（opt-in）。決定性・保存則・
境界・木性・ソフト標高忌避を保つ。**新モデル内の力学変更**（現行 Jones とは無関係）。

## 実装方針
- **パラメータ**（`TreeParams` に追加・**既定は現行挙動**）:
  - `w_rand: f64`（既定 **0.0** = ランダム探索なし＝現行挙動）。>0 で探索の強さ。
  - 必要なら `explore_persistence`（進行方向へのバイアス。純ジッタでなく滑らかな彷徨いにする）等。実装者裁量。
- **方向ブレンド**（[step.rs](../src/tree/step.rs) の tip 成長）:
  - 誘引方向（現行 `attractor_dirs`/`cluster_dirs`）に、`state.rng` から引いた**ランダム単位ベクトル**を
    `w_rand` 重みで合成。誘引が無い tip も `w_rand>0` なら合成方向（ランダム主体）で成長候補になる。
  - **決定性**: 乱数は `state.rng`（単一シードPRNG・現状未使用で保持済み）のみ。ノード/砂糖 id 昇順を維持。
- **予算消費（保存則の要）**: ランダム伸長も `b_free` を消費し、`b_free` 不足なら伸ばさない（棄却）。
  砂糖ゼロでも初期予算 `initial_budget`（＋回収分）の範囲で伸び、**尽きたら成長停止**。
- **探索 vs 退縮の共存**: `w_rand>0` のとき、誘引の無い tip は「退縮」ではなく「予算がある限り探索伸長」に
  切り替える（予算が尽きれば伸びない＝プラトー）。`w_rand=0`（既定）では現行の退縮のまま。
  ※病的な伸縮振動を避ける設計にする（実装者裁量。例: 予算しきい/非生産tip の扱い）。
- **境界/標高**: ランダム方向でも海/範囲外は棄却、標高が高いほどコスト大（ソフト忌避）を維持。

## acceptance_test（headless・複数シード中央値・S9 部分集合 [1,42,1337]）
1. **砂糖なし探索（新挙動）**: 砂糖を1つも置かず `w_rand>0` で回すと、T tick 後の**ノード数/構造長が
   初期(根のみ)より増える**（＝食べなくてもランダムに伸びる）。かつ `w_rand=0` の同条件（伸びない）より
   **有意に大きい**。中央値で判定。
2. **予算で頭打ち（保存則）**: 砂糖なし探索は**予算で bounded** — 大きな T でも構造長が
   `initial_budget` 由来の上限内に収まりプラトーする（無限成長しない）。かつ全 tick で
   `total_volume == collected − consumed`・非負・`b_free` 超過配分なし。
3. **近くの砂糖へ誘引が支配**: `w_rand>0` でも、tip 近傍（`attract_radius` 内）に砂糖を置くと
   木が到達する（最近傍ノード ≤ `sugar_radius`、中央値）。ランダムが到達を妨げない。
4. **決定性**: 同一 (seed, script, ticks, params) → 同一 `tree_state_hash`（ランダムは `state.rng` 由来）。
5. **不変条件**: 有限性・境界（全ノード陸内＝ランダム伸長の海方向は棄却）・木性（根1・親高々1・閉路なし
   連結）・ソフト標高忌避（探索でも高標高帯ノード密度 < 低標高帯、または高標高への伸長が有意に抑制）。
6. **リグレッション（最重要・既定オフ）**: `w_rand=0`（既定）で `tests/tree_growth_001.rs` の全6件が
   **緑・挙動不変**（`tree_state_hash` バイト不変）。既存の Jones 全テストも不変で緑。
   もし既定で tree_growth_001 の hash が変わる/落ちるなら、**テストを弱めず・停止して §8 で報告**。

  規約・設計軸（§0）。
- **決定性**: `state.rng` 単一・ノード/砂糖 id 昇順を崩さない。**ソフトのみ**（ランダム伸長も海棄却・標高コスト）。
- **保存則を壊さない**: ランダム伸長は必ず `b_free` を消費（無からの成長禁止）。過学習禁止（複数シード中央値）。
- 探索用一時テストは決定後に削除。§7: 通すために tick/seed/解像度を都合よく縮小しない。
