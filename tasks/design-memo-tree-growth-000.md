## 1. モデルの骨格（両案 共通）

### 1.1 状態（State）
- **ノード** `Node { id, parent: Option<id>, x, y, vol }`。ノードは木を成す（根＝コロニー種、ホーム座標）。
- **パス（辺）** = 各ノード→親の1本。パスの**距離** `d_i = |pos_i - pos_parent|`（または明示保持）。
  「親と子だけが繋がる」＝**厳密な木（閉路なし）**。
- **体積 V**（保存量）: どこが持つかは案で分岐（§2）。砂糖回収で増え、伸長・分岐で消費。
- 場は現行と共有: 陸海マスク・標高場 E・砂糖源（動詞は §0 のまま「置く/取る・時間」）。

### 1.2 遷移 `step(state, inputs, params, rng) -> state'`（決定論）
毎tick、正準順序（id 昇順、または根からのBFS順）で:
1. **回収**: 砂糖源の半径内にあるノードが砂糖を回収 → 体積 V を加算（保存台帳 collected へ）。
2. **配分**: V を「どのパスをどれだけ伸ばすか／どこで分岐するか」に分配（§2 の案が規定）。
3. **伸長**: パス i に ΔV を配ると距離が Δd = ΔV / cost(i) 増え、子ノードが親から外へ Δd 動く
   （方向は §2）。**負の配分＝退縮**（Δd<0 でノードが親へ寄る）も許すと「伸び縮み」が出る。
4. **分岐**: 条件成立でノードを新規生成（親の位置＋方向に子を1つ）。生成コスト c_branch を V から引く。
5. **境界/忌避**: 伸長先が海/範囲外なら棄却（境界不変条件）。標高が高いほど cost 増（ソフト忌避④相当）。

### 1.3 決定性・不変条件（現行契約を踏襲）
- 単一シード PRNG のみ。壁時計・ハッシュ反復順を遷移に使わない。ノード反復は id 昇順で正準化。
- **保存則**: `V_total = collected − consumed`。伸長・分岐で消費した分だけ減り、負にならない・自然増殖しない。
- **境界**: 全ノードが陸・範囲内。**有限性**: 座標/距離/体積に NaN/Inf 無し。
- **再現性**: 同一 (seed, 入力列, params) → 同一 final_state_hash。
- **ソフト標高忌避**: 高標高への伸長/分岐コストを上げる（壁は作らない＝低確率で滲める）。

## 2. 未決の核心を2案に具体化（ここを選ぶ）

「体積をどこが持ち、どう配分し、どちらへ伸ばし、いつ分岐するか」＝モデルの面白さの正体。2案示す。

### 案 α — 全体予算 ＋ 砂糖への space colonization（配向的・制御しやすい）
植物の脈/樹木生成で使われる space-colonization を土台にする。
- **体積の所在**: コロニー**全体で1つの予算 B**。回収で増える。
- **アトラクタ**: 各砂糖源＝誘引点。毎tick、各誘引点について**最近傍の木ノード**を求め、そのノードを
  誘引点方向へ伸ばす候補にする（複数誘引点が同一ノードを引くと平均方向）。
- **配分**: B の一定割合を、候補ノード群へ「誘引の強さ（近さ×残量）」で按分。ノード i の取り分 ΔV_i を
  距離に変換（Δd_i = ΔV_i / cost）してそのノード（＝パス）を誘引方向へ伸長。
- **分岐**: あるノードに**角度差の大きい2つの誘引**がかかる、またはパス長が閾値超で、子を1本spawn
  （枝分かれ）。生成は B から c_branch を消費。
- **退縮**: 残量0の砂糖＝誘引消滅。誘引を失った末端は「維持コスト」で毎tick 微小に Δd<0 退縮。
- **特徴**: 「砂糖へ伸びて枝分かれし、餌が尽きると縮む」が**規則で直接**出る。決定的（誘引点＋タイ
  ブレークで方向確定）。visually クリーンなニューロン/樹木。**創発は弱め**（形は誘引配置で決まる）。

### 案 β — ノード単位の体積 ＋ 木内フロー（創発的・調整は難しめ）
体積を木の中を流れる"圧力/樹液"として扱う。
- **体積の所在**: **各ノード `vol_i`**。砂糖回収は近傍ノードの vol を増やす。
- **フロー**: 毎tick、親子間で vol を拡散（高い方から低い方へ一定率）。根や砂糖直下で高く、末端で低い勾配。
- **成長**: ノードの vol が閾値超で、**局所の砂糖勾配＋進行方向の persistence（コーン内）**へ伸長
  （Δd = 超過分/cost）。さらに超過が大きいと子をspawn（分岐）。伸長/分岐で vol を消費。
- **退縮**: vol が枯れた末端は Δd<0 で縮む（フローが来なくなった枝が自然に引く）。
- **特徴**: 成長圧が構造を伝播する**創発的**な動き（Physarum の樹液流に近い）。**面白いが調整が難しく**、
  フロー率・閾値・コーン角の設計で挙動が大きく変わる。

### 2案の比較（判断材料）
| 観点 | α 全体予算+space colonization | β ノード体積+フロー |
|---|---|---|
| 体積の所在 | 全体1予算 | ノード単位 |
| 配向 | 砂糖アトラクタへ直接 | 局所勾配＋persistence |
| 制御性 | 高い（狙った形になりやすい） | 低い（創発で振れる） |
| 創発感 | 弱め | 強め |
| 決定論 | 容易 | 容易（フローも決定的） |
| 実装難度 | 低〜中 | 中〜高 |
| 最初の試作向き | ◎ | △（αで土台を作った後が良い） |

**推奨**: まず **α** で最小プロトタイプ（伸長＋分岐＋保存則＋砂糖配向＋退縮を各1つ）を回し、手応えを見て
から β 的な創発要素を足す/切り替える。α は「思いついてない具体ロジック」を最短で具体挙動にできる。

## 4. 最小プロトタイプの受け入れ条件（案・将来 B/実装時に人間が確定）
α を試作する場合の headless 判定可能な条件の叩き台（**本メモでは決めない**）:
- ホーム根から、距離 D に置いた砂糖へ木が**到達**（末端ノードが砂糖半径内）：複数シード中央値で成立。
- **保存則**: `V == collected − consumed`、非負。伸長・分岐の総コストと一致。
- **分岐**: 2つの砂糖を離して置くと**枝分かれ**して両方に到達（分岐ノード数 ≥ 1）。
- **退縮**: 到達後に砂糖除去 → 末端パス距離が有意に縮む（時間比較）。
- **不変条件**: 有限性・境界（陸内）・再現性（同一seed→同一hash）・ソフト標高忌避（高標高帯の
  ノード密度 < 低標高帯）。
- 決定性: 単一PRNG・ノードid昇順。
