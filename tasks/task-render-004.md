## 背景
core-002 で「ホーム凝集スタート＋誘引物質勾配コホージョンによる foraging（伸び縮み）」を
決定論コアに実装し headless で検証したが、既定オフのため**対話デモは従来の一様散布のまま**。
ユーザーが「デモに載せて実際に動かせるように」と要望（2026-07-07）。

## task_id
`render-004`

## goal
ブラウザ対話デモ（`docs/demo/`）で採餌モードを選べるようにし、
群れがホームに凝集して始まり、離れた場所に砂糖を置くと触手が伸びて到達（伸び）、
砂糖を除去すると退縮する（縮み）様子をライブで見られるようにする。
core・既存の受け入れテストは不変（core ← render の一方向依存を維持）。

## acceptance_test
1. `render-wasm` に採餌用コンストラクタ（`Sim::new_forage`）とホーム座標アクセサを追加し、
   native test で「採餌初期化は決定的」「初期エージェントがホーム近傍に凝集」「同一操作列→同一hash」を検証。
2. デモに採餌/従来トグル（既定=採餌ON）とホーム印を追加。既存の density⇔graph トグル・
   砂糖の置く/取るは従来どおり機能。
3. ブラウザ実測: 採餌モードで (a) 群れがホームに凝集、(b) 離れた砂糖へ触手が伸びて到達、
   (c) 砂糖除去でトンネルが退縮、をコンソールエラー無しで確認。
4. リグレッション: core + render-wasm の全テストが緑。wasm/JS glue を再生成しライブ整合。

## constraints
- **編集可**: `render-wasm/src/lib.rs`（読み取りのみの露出）、`docs/demo/index.html`、
  生成物（wasm/js glue）。
- **編集不可（人間所有・§7/§11）**: core の不変条件・ゴールデン・受け入れテスト、設計軸（§0）。
- **§0 の動詞は不変**: プレイヤーの動詞は砂糖 置く/取る・時間のみ。採餌モードは初期条件の
  切替であって新しい動詞ではない（クリックでコロニー配置等はしない）。
- **決定性**: render は State を読むだけ。core の挙動を変えない。

## このタスクでやらないこと
- 採餌モードを headless の既定にすること（既定オフのまま。既定化は別の明示的ゴールデン更新）。
- パラメータ（sigma/cohesion）の UI スライダー化。まずは固定プリセットで「見せる」。
