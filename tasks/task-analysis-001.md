## task_id
`analysis-001`

## goal
Jones の trail マップから効率ネットワークを導出し、その効率をメトリクスとして出力する
静的解析レイヤ `analysis/` を実装する。処理は
**しきい化 → 骨格抽出(スケルトン化) → グラフ化 → 流れを1回解く → 指標算出**。
砂糖源を流れの吸込口／湧出口として扱う。適応ループは実装しない。

## acceptance_test
headless で判定可能な条件（core-000 の `run_headless` 出力 State に対して実行）:

1. **決定性/一意性**: 同一 `state_hash` の State から解析を2回実行 →
   ネットワーク指標が**完全一致**する（3シード: 1, 42, 1337 の各終端 State で）。
2. **グラフ健全性**: 抽出グラフが well-formed
   （孤立点・自己ループ・重複辺の扱いが定義通り、ノード/エッジ数が有限、NaN/Inf なし）。
3. **coreとの整合**: 解析グラフの連結成分数が、core の `num_cc`（同一しきい値時）と整合する。
4. **忌避のネットワーク側確認**: `edge_mean_elevation`（エッジ加重平均標高）が、
   陸地全体の平均標高より低い ＝ ソフト標高忌避がネットワークレベルでも効いている。
5. **出力**: 下記メトリクスを含む `analysis.json` を出力する。
6. **非侵襲**: 解析実行前後で State と `state_hash` が不変（Stateを書き換えていない）。

## 算出するメトリクス（analysis.json）
- `nodes`, `edges`: グラフ規模
- `total_length`: エッジ総延長
- `mst_length`: 最小全域木の総延長
- `redundancy = total_length / mst_length`: 冗長度（耐障害性とコストのトレードオフ指標）
- `total_conductance`（または砂糖源間の `effective_resistance`）: 流れやすさ
- `transport_efficiency`: 幹線への集約度（例: 流量上位エッジへの流量集中度）
- `edge_mean_elevation`: エッジ加重平均標高（忌避の健康診断）
- `num_cc`, `largest_cc`: 連結成分（coreメトリクスとの照合用）

## constraints
- **core を変更しない。** `analysis/` は core を読むだけ・逆依存禁止（設計メモ §2）。
- Stateは読み取り専用。書き換え・副作用なし。
- しきい値・骨格抽出パラメータは params に集約（ハードコード禁止、規約準拠）。
- グラフ化の際、**ノード = 骨格の分岐点・端点・砂糖源**、**エッジ = 骨格の枝**。
  エッジ長は実距離、必要なら標高で加重した実効長 `L_eff = L·(1+α·平均標高)` を別途持ってよい。
- 流れは砂糖源を source/sink とした線形系（ポアソン/キルヒホッフ）を**1回だけ**解く。反復なし。
- 規約 §7（完整性）順守: しきい値を通すために縮小したり、指標を都合よく丸めない。
- 迷い・仕様の矛盾があれば実装を進めず、規約 §8 に従い停止して質問する。
