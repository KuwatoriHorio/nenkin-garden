# タスク: test-harness-001 — 正準シード集合と中央値ゲートをテストへ反映

`loop-engineering-rules-v0.md` §11 のフォーマットに沿ったタスク文。
**前提**: `calib-001` で正準シード集合 S（奇数9本）と中央値集計法が確定済み。

---

## 位置づけ
現在の受け入れ/不変条件テストはシードを `[1, 42, 1337]`（3本）で固定している。
`calib-001` で規約 §4 に確定した **S = {1, 7, 13, 42, 99, 256, 1337, 2024, 31337}**（9本）と
**S 上の中央値による方向つきソフトゲート**に、テストコード側を整合させる。

## task_id
`test-harness-001`

## goal
不変条件（Tier 0）を S 全9本で検証し、メトリクスのソフトゲート（Tier 2, §4）を
S 上の中央値・方向つき比較・許容%で判定する共通ヘルパをテストに導入する。
既存の受け入れ意図を**弱めずに**シード被覆と集計法を規約に一致させる。

## acceptance_test（headless で判定可能）
1. 不変条件テスト（有限性/保存則/境界/再現性/ソフト標高忌避）が **S 全9本**で pass。
2. `final_state_hash` の再現性（2回一致）が S の各シードで成立。
3. メトリクス・ソフトゲートを判定する共通ヘルパが存在し、S 上の中央値・方向つき比較・
   §4 の許容%（被覆−8/砂糖−18/max_cc−18/num_cc+10warn/mean_trail_lo−12/tick+25）で
   基準比較する（baseline は固定の参照値をテスト内定数として持つ）。
4. `core-000`/`analysis-001` の既存アサートは**意味を弱めない**（削除・緩和なし）。
   シード本数を増やしても既存の各不変条件は全て保持。

## constraints
- **§7 完整性**: シード被覆を規約に合わせて広げるのは可。ただし既存アサートの削除・緩和、
  合格のためのしきい緩和は禁止。テストは事実を検証する側に置く。
- 正準シード集合・許容%・集計法は**規約 §4 と一致**させる（規約が正）。
- 決定性契約（§2）を崩さない。`analysis` の core 非侵襲を崩さない。
- baseline（前回green中央値）の持ち方は裁量。ただし「意図した挙動変更」時のみ理由付きで更新。

## seeds
`{1, 7, 13, 42, 99, 256, 1337, 2024, 31337}`（規約 §4 の正準集合）。

## このタスクで意図的に「やらない」こと
- メトリクスしきい（θ_cov 等）の再調整（別タスク `metric-thresholds-001`）。
- core パラメータ既定値の変更（別タスク `core-001` 等）。
