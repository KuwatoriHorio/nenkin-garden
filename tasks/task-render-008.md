## task_id

## goal
デモに**アメーバ表現**（柔らかい発光ブロブが融合して流動的な膜＝原形質のように見える描画）を
追加し、エージェント表現を**樹状(ニューロン)／アメーバ／点**の3択で切り替えられるようにする。
**core・render-wasm の実装は変更しない**（アメーバ描画は既存の `agent_positions()` だけで描ける
＝デモ HTML/JS のみの変更。wasm 再生成も不要）。

## 実装方針（docs/demo/index.html のみ）
- **アメーバ描画** `drawAgentsAsAmoeba()`: `agent_positions()` の各エージェントを、
  **柔らかい放射状グラデーションのブロブ**（中心が明るく縁が透明・半径は数セル分）として
  `ctx.globalCompositeOperation='lighter'`（加算合成）で描く。重なったブロブが融合して
  連続した発光する膜（メタボール様）に見える。色は有機的なシアン〜緑系。必要なら中心に小さな
  明点を足す。State は読むだけ・非侵襲（描画のみ）。
- **3択トグル**: 現在の 2値 `neuronStyle`（boolean）を 3状態 `agentStyle ∈ {'neuron','amoeba','dots'}`
  に拡張。`#agentStyle` ボタンを**巡回**させる（例: 樹状 → アメーバ → 点 → 樹状…）。
  ラベルは現在の表現が分かる表記（例「🧬 表現: 樹状 / アメーバ / 点」）。
  ディスパッチャ `drawAgents(ts)` が `agentStyle` で 3つの描画関数を切り替える。
  既定は現行どおり **樹状(neuron)**（変更で驚かせない）。
- 既存機能（樹状=`drawAgentsAsNeurons`／点=`drawAgentsAsDots`、砂糖 置く/取る・density⇔graph・
  trail表示・採餌・strength/collect_rate/trail_max スライダー・速度・reset）は**従来どおり**動くこと。
- 性能: アメーバのブロブ描画はエージェント数分の放射状グラデーション。数が多いと重いので、
  半径・α を控えめにし、必要なら簡易な間引き/キャッシュで実用フレームレートを保つ
  （決定性は描画のみで影響なし）。

## acceptance_test

2. **アメーバの見た目**: エージェントが融合した発光する膜（原形質様）として描かれ、採餌で
   その膜が伸びて広がる。
3. **ブラウザ実測**: 3表現すべてが描画され、切替が機能し、コンソールエラー無し・過負荷なし。
4. **リグレッション**: core + render-wasm 全テスト緑（本タスクはデモ HTML のみ変更で、
   render-wasm/core は無変更のはず。もし render-wasm を変えたなら再生成＋native test）。

## constraints

## このタスクでやらないこと

- ピクセル閾値による厳密なメタボール輪郭抽出（getImageData 多用で重い）。まずは加算ブロブの
  融合で"にじむ膜"を表現。必要なら後続で高度化。
