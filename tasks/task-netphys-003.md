## 背景（なぜこのタスクか）
ユーザー観察（2026-07-12・demo-net seed 42）: **網が山を登る**。調査の結論:

- 現状 `netphys_step` の標高忌避は **コスト/実効抵抗のみ**で実装されている:
  - 伸長コスト割増 `tax = structural · c_elev · e_target`（`c_elev=1.5`）。予算不足なら伸長棄却
    （[src/netphys/step.rs](../src/netphys/step.rs) の Phase1 探索）。
  - Kirchhoff の実効長 `L_eff = L·(1 + net_alpha·meanE)` で高標高の管は流れにくく consolidation で先に刈られる。
- **決定的な欠落**: 探索の**進行方向**（probe 方向）は `w_rand` ランダム＋誘引だけで決まり、**標高を全く見ていない**。
  標高は「その一歩を払えるか」をゲートするだけ。よって網は**山を回り込まず**、予算が余っていれば普通に登る。
- netphys-001 の⑤標高忌避テストは `initial_budget=20`（**予算欠乏 regime**）でのみ検証しており、そこではコストが
  律速して「高標高側の構造 < 低標高側」が成立し緑。しかし netphys-002 が `initial_budget` を 400→**1200**（潤沢）へ
  上げたため、demo regime ではコストが律速せず**忌避が実質不活性 → 登る**。

これは §7 のごまかしでも invariant 違反でもない（ソフト＝壁を作らない設計は保たれている）。だが「低地を好む」という
**設計軸 §0 のソフト標高忌避が、通常予算では視認できるほど効いていない**。本タスクは忌避を**方向で**効かせ、
**通常予算でも低地選好が観測できる**ようにする。壁は作らない（確率的には登れる）。

**追加要望（2026-07-12・ユーザー）**: 現状の探索は**全体として「線が伸びる」**ように見える（各前線ノードが
持続方向へ 1本ずつ伸長）。探索時は**周囲へ扇状に拡散して広がってほしい**（面的に開く＝実際の Physarum の
「数箇所から扇状に広がる」に近づける）。**ノード数が増えてもよい**。まず**この探索方法に変えて挙動を観察したい
（試行）**。＝本タスクは exploratory: 実装後に demo-net へ反映して**目視で挙動を確認**し、その結果を見て
既定値の採否・微調整を人間が判断する。

## task_id
`netphys-003`

## goal
`netphys_step` の **Phase1 探索を「扇状拡散＋標高方向バイアス」に変える**:
- **(A) 扇状拡散**: 各前線ノードから**単一方向に線を伸ばす**のでなく、**周囲の角度範囲へ複数の probe を扇状に**
  広げて枝分かれ・面的に開くようにする（ノード数増を許容）。
- **(B) 標高方向バイアス**: probe 方向を**低標高側へ確率的に寄せる**（`−w_elev·∇E` 相当）。
両方を入れて、**通常予算 regime（`NetParams::default()`, initial_budget=1200）でも
①コロニーが低標高帯に偏り②探索が面的に広がる（線でなく扇状）**ことを headless で実証し、demo-net で目視確認する。
ソフト（壁なし・確率的に登れる）を保ち、netphys-001 ①②⑤⑥・netphys-002 ③④・既存 Jones/tree/analysis 全テストを
**緑のまま**にする。

## 実装方針
- **(A) 扇状拡散**（`src/netphys/step.rs` Phase1・`src/netphys/state.rs` の `NetParams`）:
  - 前線ノードの伸長を、**持続方向±角度範囲に複数 probe を張る扇状探索**へ変える（生存 probe ごとにノード/辺を
    追加＝枝分かれ）。1本しか通らない場合は従来どおり単枝。**面的に開く**ことを狙う。
  - 新規パラメータ例 `fan_count`（1本あたりの probe 本数）・`fan_spread`（扇の半角）・分岐確率。既定は**扇状に
    広がる妥当値**を探索用一時テストで実測。既定を保守値にすれば現状に縮退できる（後方互換のフォールバック）。
  - **ノード数増を許容**: 必要なら `node_cap`/`edge_cap` の既定を上げてよい（＝設計判断であり、テストを通すための
    水増しではない・§7）。ただし**⑥有界は維持**（cap 内・爆発しない・周期 Kirchhoff が予算内）。
- **(B) 標高方向バイアス**: probe 方向の選定/重みに標高勾配 ∇E を乗せ、**低標高方向を確率的に優先**する
  （例: 方向重み `∝ exp(−w_elev·(e_probe − e_here))`）。決定的・単一 PRNG・id 昇順を厳守。
  - 既存の**コスト割増 `c_elev`・実効抵抗 `net_alpha` はそのまま残す**（方向バイアスは加算的な第3の忌避経路）。
  - `w_elev` 既定は通常予算でも忌避が観測できる妥当値を**探索用一時テストで実測**。0 なら方向バイアス無しに縮退。
- **壁禁止（§0）**: (A)(B) いずれもソフト。高標高セルへの伸長を**禁止しない**（確率を下げるだけ・稀に登れる）。
  海/範囲外の棄却は従来どおり（ハード境界で標高とは別軸）。
- **保存則・決定性を壊さない**: 方向選択・分岐数の変更のみで質量会計（free↔D·L・回収/消費/刈り込み）は不変。
  扇状分岐・確率選択でも `total_mass == collected − consumed` を締める（⑤が引き続き検証）。

## acceptance_test（headless・複数シード中央値・S9 部分集合 [1,42,1337]・新規 `tests/netphys_003.rs`）
1. **通常予算での低地選好（主判定・B）**: `NetParams::default()`（initial_budget=1200・砂糖なし
   または対称配置）で K サイクル回した最終状態で、**高標高帯のノード密度 < 低標高帯のノード密度**
   （または「ノードの標高加重平均 < 陸地平均標高」）が**中央値で有意なマージン**をもって成立する。
   ＝ netphys-001 ⑤の scarce regime だけでなく **通常予算でも**忌避が観測できること。
   - baseline 比較で締める（§7 exemplary を推奨）: `w_elev=0`（方向バイアス無し＝現状）では通常予算で
     低地選好が成立**しない**ことを**赤**として確認 → `w_elev`（既定・効く値）で**緑**。同一 world・同一 seed で対比。
2. **扇状拡散＝面的に広がる（A）**: 固定 tick 時点で、探索が**線状でなく面的に開く**ことを定量で締める。
   例: 初期数ノードの**方向分散（角度カバレッジ）／占有グリッドセル数／凸包面積**が、扇状無効（`fan_count=1`＝現状の
   線的伸長）baseline より**中央値で有意に大きい**。＝ red（線）→ green（扇状）で対比（§7 exemplary）。
3. **ソフト性（壁でない）**: 十分なサイクルで、コロニーは**高標高セルにもノードを持ちうる**（完全排除でない）。
   ＝バイアスは確率的で、登攀を全面禁止していないこと（急峻地形でも稀に到達しうる）を弱い形で確認。
- **回帰（必須）**: `tests/netphys_001.rs` ①②⑤⑥・`tests/netphys_002.rs` ③④が**緑のまま**。
  特に ⑤（scarce regime の忌避）・③（前進波・free_budget>50）・④（効率改善）・⑥（有界＝cap 内）を壊さないこと。
  Jones/tree/analysis/render-wasm 全テストも緑。**決定性 hash 契約**を壊さない（同一 seed・入力→同一 `netphys_state_hash`）。
- パラメータ（`w_elev`・`fan_count`・`fan_spread` 等）は**探索用一時テスト**で実測 → 決定後に削除。
  まず①（w_elev=0 で低地選好せず）・②（fan_count=1 で線的）の赤を確認してから実装。
- **試行としての目視確認（合否外だが必須の締め）**: 全テスト緑を確認後、**demo-net の wasm を再生成**して
  ライブで「扇状に広がる／低地に寄る」挙動を人間が目視確認できる状態にする（デモの JS 描画ロジックや
  netphys 以外は無変更・wasm 再生成のみ）。目視結果を見て既定値の採否・微調整を人間が判断する。

## §8 停止の指針（このタスク特有）
扇状拡散・方向バイアスを効かせると他の性質と綱引きになりうる。次に当たったら**§7 のごまかしをせず停止**し、
成立した部分と詰まり・選択肢を報告する:
- 低地選好を強くすると②餌連結（高標高側の砂糖に届かない）や①網化が壊れる（忌避 vs 到達の綱引き）。
- 前進波③が止まる／効率④が悪化する（方向拘束で外向き変位が出ない）。
- 通常予算で①（低地選好）を満たす `w_elev` の妥当帯が無い（弱すぎて登る／強すぎて壁化＝ソフト性③を破る）。
- 扇状化でノード/辺が cap を突く・爆発する／周期 Kirchhoff が予算超過（⑥有界 vs 面的拡散の綱引き）。
無理に①②を通すために cap/tick を縮小したり（cap 引き上げは可・水増しは不可）、netphys-001/002 の受け入れを
弱めたりしない。設計軸（§0 ソフト忌避）自体を変えたくなったら §8 で停止・相談。
