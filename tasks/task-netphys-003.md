## 背景（なぜこのタスクか）
ユーザー観察（2026-07-12・demo-net seed 42）: **網が山を登る**。調査の結論:

- 現状 `netphys_step` の標高忌避は **コスト/実効抵抗のみ**で実装されている:
  - 伸長コスト割増 `tax = structural · c_elev · e_target`（`c_elev=1.5`）。予算不足なら伸長棄却
    （[src/netphys/step.rs](../src/netphys/step.rs) の Phase1 探索）。
  - Kirchhoff の実効長 `L_eff = L·(1 + net_alpha·meanE)` で高標高の管は流れにくく consolidation で先に刈られる。
- **決定的な欠落**: 探索の**進行方向**（probe 方向）は `w_rand` ランダム＋誘引だけで決まり、**標高を全く見ていない**。
  標高は「その一歩を払えるか」をゲートするだけ。よって網は**山を回り込まず**、予算が余っていれば普通に登る。
- netphys-001 の⑤標高忌避テストは `initial_budget=20`（**予算欠乏 regime**）でのみ検証しており、そこではコストが
  律速して「高標高側の構造 < 低標高側」が成立し緑。しかし netphys-002 が `initial_budget` を 400→**1200**（潤沢）へ
  上げたため、demo regime ではコストが律速せず**忌避が実質不活性 → 登る**。

これは §7 のごまかしでも invariant 違反でもない（ソフト＝壁を作らない設計は保たれている）。だが「低地を好む」という
**設計軸 §0 のソフト標高忌避が、通常予算では視認できるほど効いていない**。本タスクは忌避を**方向で**効かせ、
**通常予算でも低地選好が観測できる**ようにする。壁は作らない（確率的には登れる）。

## task_id
`netphys-003`

## goal
`netphys_step` の Phase1 探索に**標高の方向バイアス**（低標高方向へ probe を寄せる／高標高方向の重みを下げる、
`−w_e·∇E` 相当）を加え、**通常予算 regime（`NetParams::default()`, initial_budget=1200）でも
コロニーが低標高帯に偏る**ことを headless で実証する。ソフト（壁なし・確率的に登れる）を保ち、
netphys-001 ①②⑤⑥・netphys-002 ③④・既存 Jones/tree/analysis 全テストを**緑のまま**にする。

## 実装方針
- **方向バイアスを Phase1 探索に加える**（`src/netphys/step.rs`・`src/netphys/state.rs` の `NetParams`）:
  - probe 方向を選ぶ際、候補方向ごとに標高勾配 ∇E を評価し、**低標高方向を確率的に優先**する
    （例: 方向重み `∝ exp(−w_elev · (e_probe − e_here))` を w_rand ランダム/誘引に乗せる、または
    複数候補 probe から低標高側を確率選択）。決定的・単一 PRNG・id 昇順を厳守。
  - 既存の**コスト割増 `c_elev` と実効抵抗 `net_alpha` はそのまま残す**（方向バイアスは加算的な第3の忌避経路）。
  - 新規パラメータ `w_elev`（方向バイアス強度・既定は忌避が通常予算でも観測できる妥当値を**探索用一時テストで実測**）。
    既定 0 なら現状と完全一致（＝後方互換のフォールバック）だが、本タスクの狙いは既定を**効く値**にすること。
- **壁禁止（§0）**: バイアスはソフト。高標高セルへの伸長を**禁止しない**（確率を下げるだけ・急峻でも稀に登れる）。
  海/範囲外の棄却は従来どおり（これはハード境界で標高とは別軸）。
- **保存則・決定性を壊さない**: 方向選択の変更のみで、質量会計（free↔D·L・回収/消費/刈り込み）は不変。
  非放射・確率選択でも `total_mass == collected − consumed` を締める（⑤が引き続き検証）。

## acceptance_test（headless・複数シード中央値・S9 部分集合 [1,42,1337]・新規 `tests/netphys_003.rs`）
1. **通常予算での低地選好（本タスクの主判定）**: `NetParams::default()`（initial_budget=1200・砂糖なし
   または対称配置）で K サイクル回した最終状態で、**高標高帯のノード密度 < 低標高帯のノード密度**
   （または「ノードの標高加重平均 < 陸地平均標高」）が**中央値で有意なマージン**をもって成立する。
   ＝ netphys-001 ⑤の scarce regime だけでなく **通常予算でも**忌避が観測できること。
   - baseline 比較で締める（§7 exemplary を推奨）: `w_elev=0`（方向バイアス無し＝現状）では通常予算で
     低地選好が成立**しない**ことを**赤**として確認 → `w_elev`（既定・効く値）で**緑**。同一 world・同一 seed で対比。
2. **ソフト性（壁でない）**: 十分なサイクルで、コロニーは**高標高セルにもノードを持ちうる**（完全排除でない）。
   ＝バイアスは確率的で、登攀を全面禁止していないこと（急峻地形でも稀に到達しうる）を弱い形で確認。
- **回帰（必須）**: `tests/netphys_001.rs` ①②⑤⑥・`tests/netphys_002.rs` ③④が**緑のまま**。
  特に ⑤（scarce regime の忌避）・③（前進波・free_budget>50）・④（効率改善）を壊さないこと。
  Jones/tree/analysis/render-wasm 全テストも緑。**決定性 hash 契約**を壊さない（同一 seed・入力→同一 `netphys_state_hash`）。
- パラメータ（`w_elev` 等）は**探索用一時テスト**で実測 → 決定後に削除。まず①の赤（w_elev=0）を確認してから実装。

  `test_harness_001.rs`/`metric_thresholds_001.rs` の baseline、analysis 受け入れ、規約・設計軸（§0）。
  **Jones/tree のモデルコードは変更しない**。
- **決定性**: 方向選択は単一 PRNG・id 昇順。壁時計・ハッシュ反復順を使わない。
- **ソフトのみ・壁禁止（§0）**: 標高忌避は確率バイアス＋コスト＋実効抵抗で表現し、高標高への伸長を絶対禁止しない。
- **過学習禁止**: 単一シードだけで低地選好する調整は不可。複数シード中央値。§7: 通すために tick/seed/cap を都合よく縮小しない。

## §8 停止の指針（このタスク特有）
方向バイアスを効かせると他の性質と綱引きになりうる。次に当たったら**§7 のごまかしをせず停止**し、
成立した部分と詰まり・選択肢を報告する:
- 低地選好を強くすると②餌連結（高標高側の砂糖に届かない）や①網化が壊れる（忌避 vs 到達の綱引き）。
- 前進波③が止まる／効率④が悪化する（方向拘束で外向き変位が出ない）。
- 通常予算で①主判定を満たす `w_elev` の妥当帯が無い（弱すぎて登る／強すぎて壁化＝ソフト性②を破る）。
無理に①を通すために cap/tick を縮小したり、netphys-001/002 の受け入れを弱めたりしない。設計軸（§0 ソフト忌避）自体を
変えたくなったら §8 で停止・相談。
