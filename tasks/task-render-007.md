## 背景
現状デモはエージェントを独立した白い点で描く（render-005 `drawAgents`）。
ユーザー要望（2026-07-07）: それぞれのエージェントが**有機的につながって見える**ようにしたい。
アメーバ／ニューロンのような**樹状**で、枝を伸ばして広がっていく見え方。

## task_id
`render-007`

## goal
エージェントの表現を「点の集合」から「**近接エージェントを枝でつないだ有機的な樹状構造**」へ変更する。
採餌の触手が伸びるにつれ枝が伸長・分岐して広がる見え方にする。**core の力学は変更しない**
（render は State を読むだけ・core←render 一方向依存）。

## 実装方針
- **render-wasm/src/lib.rs**: `pub fn agent_links(&self, radius: f64) -> Vec<u32>` を追加。
  各エージェントを半径 `radius` 内の近傍エージェントと結ぶリンクを flat `[a0,b0,a1,b1,...]`
  （エージェント index ペア）で返す。**グリッド空間分割**で近傍探索して O(n) 程度に抑え、
  出力は決定論的順序（a<b・index 昇順ソート等）にする。State を読むだけ・非侵襲。
  ※密なメッシュを避け樹状に見せたい場合は「各エージェント→半径内の最近傍1〜数本」に絞ってよい
  （実装者裁量。目標は"枝分かれした樹状"であって密網でない）。
- **docs/demo/index.html**: エージェント描画を樹状レンダラに差し替え。リンクを**柔らかく発光する
  半透明の枝**（加算合成 `globalCompositeOperation='lighter'` 等・太さ/α で有機的に）として描き、
  エージェントは小さな発光ノードに。既定をこの「樹状（有機）」表現とし、**従来の点表示に戻せる
  トグル**を残す。リンク再計算は throttle してよい（決定性は"同一State→同一リンク"で担保）。
- エージェント数は成長で増えうる（最大 agent_cap_max）。グリッド近傍＋throttle＋必要ならリンク数上限で
  フレーム性能を保つ。

## acceptance_test
1. **render-wasm 露出**: `agent_links(radius)` を追加。native test で
   「同一 State → 同一リンク（決定論）」「呼び出し前後で `state_hash` 不変（非侵襲）」
   「リンクの index が全て有効範囲内・a<b・自己リンク無し」を検証。
2. **デモ UI**: エージェントが樹状（枝でつながった有機的構造）で描画される。既定=樹状、
   トグルで従来の点表示。既存機能（砂糖 置く/取る・density⇔graph・trail表示・採餌・各スライダー・
   速度・reset）は従来どおり動く。
3. **ブラウザ実測**: 樹状表現が描画され、採餌で枝が伸びて広がること、点表示トグルの切替、
   をコンソールエラー無しで確認。過負荷（極端なフレーム落ち）が無いこと。
4. **リグレッション**: core + render-wasm 全テスト緑。wasm/JS glue 再生成しライブ整合。

## constraints
- **編集可**: `render-wasm/src/lib.rs`（読み取りアクセサのみ）、`docs/demo/index.html`、生成物。
- **編集不可（人間所有・§7/§11）**: core の不変条件・ゴールデン・受け入れテスト、設計軸（§0）。
- **core の力学非変更**: render は State を読むだけ。`agent_links` は State を書き換えない。
- §0 の動詞不変（表現の変更であってプレイ動詞の追加ではない）。
- 決定性: 描画は State を読むだけ。リンク幾何は「同一 State → 同一リンク」で決定的。

## このタスクでやらないこと
- エージェントの挙動（step）を変えること。見た目（render）だけを変える。
- 物理的に正しい膜/細胞モデル。あくまで近接リンク＋発光による"有機的な樹状"の表現。
