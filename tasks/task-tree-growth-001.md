## モデル仕様（設計メモ §1・§2-α に準拠）
- **状態**: ノード `{id, parent: Option<id>, x, y}`（木・親子パスのみ・閉路なし）。根＝ホーム座標
  （`world.default_home` 流用可）。パス距離 `d_i = |pos_i − pos_parent|`。全体予算 `B`（保存量）。
  砂糖源（座標・strength・remaining・回収台帳 collected/consumed）。
- **場は現行と共有**: 陸海マスク・標高場 E（`world.rs` 流用）。動詞は §0 のまま（砂糖 置く/取る・時間）。
- **`tree_step`（決定論・ノード id 昇順で処理）**:
  1. **回収**: 砂糖半径内のノードが砂糖を回収 → B 増（collected へ計上、sugar_remaining 減）。
  2. **配分（space colonization）**: 残量>0 の各砂糖＝誘引点。誘引点ごとに**最近傍の末端ノード（tip）**を
     求め、その tip を誘引方向へ伸ばす候補にする。B の一部を候補へ按分（近さ×残量で重み）。
  3. **伸長**: 候補 tip のパス距離を Δd = 配分ΔV / cost(E) 増やし、tip を誘引方向へ Δd 移動。
     **体積は保存的に再配分**: `total_volume = B_free + Σ(k·d_i)`。伸長は B_free→構造へ移すだけ。
  4. **分岐**: tip に**角度差の大きい2誘引**がかかる、またはパス長が上限超で、子ノードを1本 spawn
     （分岐）。生成コストを B_free から引く。
  5. **退縮**: 誘引を失った tip は維持コストでパス距離を毎tick 微小に縮める（Δd<0）。d≤0 で prune、
     解放分は B_free に戻す（保存的）。
  6. **境界/忌避**: 伸長先が海/範囲外なら棄却（tip は動かさない）。**標高が高いほど cost 大**
     （ソフト忌避＝壁でなくコスト増）。
- **パラメータ**は新規 `TreeParams`（現行 `Params` を汚さない）: cost_per_length k・c_branch・
  1tick 成長率・分岐の角度/長さしきい・退縮率・誘引半径・初期予算 等。既定値・具体数は実装者裁量
  （実測で調整可）。

## acceptance_test（headless で判定可能・複数シード中央値）
検証は正準9シード S9 の部分集合（最低 `[1,42,1337]`）。

1. **到達（伸び）**: ホーム根から距離 D（例 12）に砂糖1つを置くと、T tick 以内に**いずれかのノードが
   砂糖半径内へ到達**する。複数シード中央値で成立。
2. **保存則**: `total_volume == collected − consumed`（または `B_free + Σk·d_i (+分岐) == collected −
   その他消費`）が常に成立、**非負**、B_free を超える配分をしない。
3. **分岐**: 2つの砂糖を離して置くと、木が**枝分かれ**（分岐ノード数 ≥ 1）して**両方に到達**する
   （中央値）。
4. **退縮（縮み）**: 到達後に砂糖を除去すると、餌を失った末端パスの距離が**有意に縮む**（時間比較）。
5. **不変条件**: 有限性（座標/距離/B に NaN・Inf なし）、境界（全ノードが陸・範囲内）、
   再現性（同一 seed・入力列 → 同一 `tree_state_hash`）、ソフト標高忌避（高標高帯のノード密度 <
   低標高帯のノード密度、または高標高への伸長が有意に抑制される）。
6. **木性**: 閉路なし（根以外の全ノードが親をちょうど1つ持ち、根から全ノードへ到達可能＝連結な木）。
