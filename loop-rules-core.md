# 粘菌ガーデン — 常駐規範コア (v0-core)

これは毎ターン常駐する規範の**核**。全文の正典は [loop-engineering-rules-v0.md](loop-engineering-rules-v0.md)（人間所有・変更は §8 の承認タスクのみ）。
数値しきい・手順・モデル割り当ての詳細はオンデマンドで読む（§A参照）。判定の真実はコードにある。

---

## §0 プロダクト設計軸（変更不可・ドリフト禁止）

- モデル: エージェントベース（Jones模型）trailマップ + 拡散・減衰。
- ジャンル: 園芸・観察型。プレイヤーの動詞は「砂糖を置く／取り除く」「時間速度の変更」のみ。
- 場: 陸海マスク + 標高場 E（仮想日本列島）。標高忌避は**ソフト**（壁を作らない）。
- 成長ループ: 砂糖到達 → バイオマス増 → エージェント上限増 → 拡張。
- **決定論コア（シード固定・描画分離・ヘッドレス実行可）を「正」とする。**

これらは「面白さの正体」。ループが勝手に方針転換してはならない。変えたくなったら §8 で停止。

## §1 目的と原則（Prime Directive）

次の3つを同時に満たす（順序＝優先度）:
1. **不変条件を絶対に壊さない**（有限性/保存則/境界/再現性/ソフト標高忌避）。
2. **合意済みメトリクスを許容範囲を超えて悪化させない**。
3. **与えられたタスクの受け入れテストを通す**。

「良くなった」の判断はエージェントの主観ではなく、**人間が書いた受け入れテスト**が定義する。
見た目の良し悪しを自己申告して合格にすることは禁止（§7）。

## §2 決定論コアの契約（土台・「正」）

- コアは純関数 `step(state, inputs, params, rng) -> state'`。同一 (初期state, inputs列, params, seed) は**必ず同一の最終stateとハッシュ**。
- 乱数は**単一シード付きPRNGのみ**。壁時計・ハッシュ反復順など非決定的入力を状態遷移に使わない。
- 描画は state を**読むだけ**。ヘッドレス実行エントリを持つ。
- コアの決定性を壊す変更は機能追加ではなく**リグレッション＝即revert**。

## §7 完整性ルール（テストのごまかし禁止・最重要）

以下はすべて**即時失格・自動revert・人間へ報告**:
- 不変条件・ゴールデン・受け入れテスト・しきい値の**無効化・削除・緩和**。
- ゴールデン値を「意図した挙動変更」の理由なしに書き換える。
- ティック数・シード数・場の解像度を**テストを通すために**縮小する。
- 特定シードだけで良くなる変更（過学習）。必ず複数シード中央値で判定。
- 例外の握りつぶし・NaNクランプ等で不変条件チェックを実質無力化。
- 主観の自己申告で受け入れとすること。
- 性能予算のために計算をスキップして**挙動を変える**こと。

原則: **テストを事実に合わせるのではなく、事実（挙動）をテストに合わせる。**
テストが間違っていると考える場合も勝手に直さず §8 で停止・提案する。
（人間所有ファイルの編集は PreToolUse フックで物理的に拒否される — §11。）

## §8 停止条件とエスカレーション

次のいずれかで**ループを止め人間に諮る**（勝手に進めない）:
- 試行上限（N回, 詳細は §A）を使い切っても受け入れテストが通らない。
- 満たせない／矛盾する不変条件・要件に突き当たった。
- タスク仕様が曖昧で複数の妥当な解釈がある。
- 性能が予算を大きく超え、挙動を保ったままでは戻せない。
- **不変条件・ゴールデン・設計軸（§0）自体を変えたくなった**とき。
- §7 のごまかしをしないと通せない状況。

エスカレーションには「何を試したか・なぜ詰まったか・選択肢」を添える。

## §8.1 タスク作成 → 実装の人間チェックポイント（必須・恒久ルール）

タスク文（§11 形式）を切り出したら、**実装に入る前に必ず停止して人間に確認する**。
人間がタスク文を読んでから、(a) 実装を進めてよいか、(b) §12 に従いサブエージェントを起動するか
（どの工程をどのモデルへ委譲するか）を判断する。**エージェントは人間の合図を得るまで実装（受け入れテスト作成・コード変更）を開始しない。**
判断が難しい場合は人間が相談を求めてよい。タスク切り出しコミット自体は停止前に行ってよいが、その直後に確認を挟む。
（2026-07-07 追加・人間所有の恒久ルール。ユーザー指示による。）

## §11 スコープ（触ってよい / だめ）

- **編集可**: シミュレーションコア実装、描画、パラメータ既定値（受け入れテストが守る範囲で）。
- **編集不可（人間所有）**: 不変条件の定義としきい値、ゴールデンの期待値（意図変更時に理由付きでのみ）、受け入れテスト、本規約・正典・常駐コア、設計軸（§0）、モデル割り当て表。
  → これらは `.claude/protected-paths.txt` に登録され、PreToolUse フックで Edit/Write が拒否される。
- ネットワークは使わない前提。外部依存の追加は §8 で提案。
