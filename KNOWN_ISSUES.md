## ISSUE-001: 代表シナリオで2砂糖源が連結しない（本物の橋が張れない）

- **状態**: ✅ RESOLVED（2026-07-07, analysis-003 + core-001）
- **発見**: 2026-07-07
- **解決**: 二段構え。①[analysis-003](tasks/task-analysis-003.md): flow ソルバが浮遊成分で特異化し
  連結を**偽陰性**にしていた重大バグを修正（過去の 0/9 計測はこのバグ由来だった）。
  ②[core-001](tasks/task-core-001.md): ビーコンを半径3の小ブロブ化（`sugar_beacon_radius=3`）し、
  近接2源のブロブが拡散で融合。結果、代表シナリオで **S9=9/9 連結**（conductance≈0.8）。
  §7遵守: しきい恣意化なし。当初の「調整で連結不可」結論は上記ソルババグの影響による誤りだった。
- **関連タスク**:
  - [core-001（網の連結性チューニング）](tasks/task-core-001.md) — 未完・§8停止（この問題の本体）
  - [analysis-002（snap規則の修正）](tasks/task-analysis-002.md) — 完了（アーティファクトは解消）
  - [calib-001（未確定数値の確定）](tasks/task-calib-001.md) — 過学習判定に使った正準9シードの出所
  - [metric-thresholds-001](tasks/task-metric-thresholds-001.md) — θ_cc 側の対処はこちらの範囲
- **記録**: [ITERATION_LOG.md](ITERATION_LOG.md) の `iter:4`（core-001 §8停止）, `iter:5`（analysis-002）

### 症状
代表シナリオ（低標高セルに砂糖2源・距離約7.2）で、analysis の `flow_connected` が
正準9シードで **0/9〜2/9**。`total_conductance` / `effective_resistance` /
`transport_efficiency` が意味を持たない（源間に経路がないため）。

### 原因分析
1. 当初は analysis の「砂糖源→最近傍ノード」snap が**孤立ビーコンスパイク**（骨格上 size=1
   の成分）を拾うアーティファクトが原因と診断（iter:4）。
2. [analysis-002](tasks/task-analysis-002.md)（近傍への多tap＋拡張グラフでの連結判定）で
   **アーティファクトは解消**（制御テストで証明済み）。
3. しかし代表シナリオは依然連結せず（iter:5）。実測により、**本体網が sugar1 の tap 半径内に
   届いていない＝2源間の gap が実在**すると確認。snap の誤りではなく、**core が2源間に
   tube を育てられていない**のが本質。
4. パラメータ総当り（密度／拡散／ビーコン／センサ距離／ticks）では、連結は「若い網の
   一過的な橋」に依存して不安定。最良でも `[1,42,1337]` で 2/3・正準9本で **2/9**＝
   そのシードへの過学習（iter:4）。

### 却下した対処（§7 完整性ルール）
- **tap_radius の恣意的拡大** → 「見かけ上つなげる」に当たるため不可（近傍網の実接続のみ tap）。
- **θ_cc の恣意的引き下げ** → 同上。ただし分解能目的の再調整なら
  [metric-thresholds-001](tasks/task-metric-thresholds-001.md) の範囲で別途検討可。
- **overfit な 2/3 構成の出荷** → 複数シード中央値の思想（§7）に反するため不可。

### 候補（未決定・人間判断待ち）
1. **ビーコンを小半径化**（core: `src/step.rs` の beacon を1細胞→3×3 等の小ブロブに）。
   食源が周囲網に埋め込まれ、近接2源間に tube が形成・安定しやすくなる見込み。
   モデル設計軸（§0）には触れない。→ core-001 の新仮説として実装できる。
2. **「全シナリオが連結する必要はない」と割り切る**（疎な網では源間に経路が無いのは正直な
   結果）。core-001 の acceptance（人間所有）を見直す。
3. その他（走化性・持続性の調整など）。

core-001 を候補1（ビーコン小半径化）の仮説で再開するか、acceptance を見直すかを人間が決定する。
